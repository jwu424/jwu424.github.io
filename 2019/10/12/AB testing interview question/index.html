AB testing interview question
Created 2019-10-12| Updated 2019-10-25|AB Testing
Word count: 1.3k|Reading time: 7 min|Post View:

  1. What is A/B testing?

    A/B testing is a method of comparing two versions of a webpage or app against each other to determine which one performs better. It is an essentially an experiment where two or more variants of a page are shown to users at random and statistical analysis is used to determine which variation performs better for a given conversion goal.

  2. Describe the process of A/B test
    A/B test can be summarized into the 5 steps below:

    • choose and characterize metrics to evaluate your experiment, i.e. what do you care about, how do you want to measure the effect. Brain storm potential metrics. Use customer conversion funnel to summarize the process. Invariant metric does not relate to the change. Evaluation metrics are related to the change. We can also use OEC as our metrics.
    • choose significant level (alpha), statistical power (1-beta) and practical significance level you really want to launch the change if the test is statistically significant
    • Using the above information to calculate required sample size.
    • Randomly assign each user to experiment or control group. Collect the data and wait until the required sample size is reached.
    • Analyze the results and draw valid conclusions

    Sanity check:

    • invariant metric does not change in experiment and control
    • Test whether the number of observations in each group is significant or not.

    Analyze evaluation metrics:

    • Using pooled mean/conversion probability.
    • Calculating pooled standard deviation and margin of error (z*sd).
    • Calculating the CI for the diff: $p_{diff}\pm z*sd$.
    • Check whether CI includes 0. If, then retain null hypothesis and think they are no difference. Else, the difference has statistically significant.

    • Compare CI with required differences to determine whether it is practical significant.

    • Sign test: confirm the result with sign test. The number of success out of total trial is statistically significant.

  3. Situations we can’t analyze through A/B test

    • Can’t test new experience.

      • Don’t have base of your comparison
      • How much time needed for users to adapt to the new experience.

    • If need explanations.

    • Long-term effect. Since we usually just run for a couple of weeks. If look for long-term effects, then need to construct OEC, or run experiments for long and continue measuring. Both of these cases are hard actually.
    • Primacy and novelty effect. For changes that require changing user behavior (e.g., users are already used to something, or, conversely, something new flashes), you need to use caution and look for trends.

  4. When not use A/B test

    • You Don’t Yet Have Meaningful Traffic
    • You Can’t Safely Spend The Time
    • You Don’t Yet Have An Informed Hypothesis
    • There’s Low Risk To Taking Action Right Away

  5. How many variates should we have in A/B test.

    One single metric or an OEC.

    Reasons:

    • A number of factors can muddy the test result water, it will be hard to figure out which one is the truly useful.
    • Usually right now, we have enough data to analyze. It is affordable to run one first and then the others.

  6. What should I do if I don’t trust the results?

    First, we should do sanity check. That’s mean we should exclude all errors.

    Then, if everything is fine, and we still not trust the results, the best way is to run a same AB testing again. Treat it as an entirely separate test and see if you can replicate the results. If you can replicate again and again, you probably have a solid set of results.

  7. What if I do not have control?
    A control is the existing version of a landing page or webpage that you are testing against. Sometimes you may want to test two versions of a page that never existed before… and that’s oaky. Just choose one of the variations and call that one the control. Try to pick the one that’s the most similar to how you currently design pages and use the other as the treatment.

  8. When A/B test is not useful, what you can do?

    Can’t do:

    • Analyze the user activity logs
    • Conduct retrospective analysis
    • Conduct user experience research
    • Focus groups and surveys
    • Human evaluation

  9. Metrics

    There are two kind of metrics.

    • Invariant metrics. Used for sanity check. It should be the same for all groups. If not, there may a bug or mistake in our design or code.
    • Evaluation metrics. Used to determine which variant is better. Usually like CTR, DAU that measures the performance of webpages.

    There are four categories of metrics:

    • Sums and counts
    • Distribution (mean, median, percentiles)
    • Probability and rates (click through probability and click through rate)
    • Ratios: any two numbers divide by each other

    Sensitivity and robustness:

    • You want to choose a metric that has high sensitivity, so the metric can pick up the change you care about.
    • You also want the metric to be robust against changes you don’t care about.
    • There is a balance between the sensitivity and robustness, you need to look into the data to find out which metric to use.

    How to measure the sensitivity and robustness?

    • Run experiments
    • Use A/A test to see if metrics pick up difference (if yes, then the metric is not robust)
    • Retrospective analysis

  10. Significance level, statistical power and practical significance level

    • Usually the significance level is 0.05 and power is 0.8.
    • Practical significance level varies depends on the test and the metrics
    • Practical significance level is higher than statistical significance level.
    • Usually, you will not launch a change that is statistically significant but not practically significant.
    • Many cost to launch a new change: engineering cost, customer support, sales issue and opportunity cost
    • The business impact of the change

  11. How to estimate(calculate) sample size?
    Required Sample size depends on 5 parameters:

    • The conversion rate value of control variation (baseline value)

    • The minimum difference between control and experiment which is to be identified.
      The smaller the difference between experiment and control to be identified, the bigger the sample size is required.

    • Chosen confidence/significance level

    • Chosen statistical power

    • Type of the test: one or two tailed test. Sample size for two tailed test is relatively bigger.

    There are different kinds of online testing tools, G-power, Evan Miller, google analytics, etc.

  1. Correlational VS causal

    • Correlation is a statistical measure (expressed as a number) that describes the size and direction of a relationship between two or more variables. A correlation between variables, however, does not automatically mean that the change in one variable is the cause of the change in the values of the other variable.

    • Causation indicates that one event is the result of the occurrence of the other event; i.e. there is a causal relationship between the two events. This is also referred to as cause and effect.

    • If you see many people take umbrella today, then it may have a high chance to rain. This is correlation not causal. You can’t say that many people take umbrella lead to rain.

    • Best Refuted Causal Claims from Observations Studies

  2. Advantages of A/B test

    • Scientific way to prove causality, i.e. the changes in metrics are caused by changes introduced in the treatment.
    • Sensitivity: you can detect tiny changes to metrics
    • Detect unexpected consequences
